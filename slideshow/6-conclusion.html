Synthèse et Conclusion

La conclusion de ce projet qui nous a bien occupé (O)

Les points positifs

Les points que j'ai trouvé positifs :
  • Le travail en équipe qui nous a rapproché du contexte professionnel.
  • les revues de code que nous avions instaurées :
    Lorsqu'une personne envoyait une Pull Resquest, les 2 autres devait relire le code et le valider avant que le MERGE ne soit fait. (O)
    Ce qui a donné lieu a des discussions sur l'organisation, par exemple des composants, ou de l'extraction du code CSS vers les différents composants. (O)

Difficulté

Un des points qui m'a posé problème, et que j'améliorerais surement sur mes prochains projets, c'est la multitude de Users Stories.
J'ai voulu qu'une User Storie soit la plus fine possible en granularité.
Pour un CRUD, par exemple, j'ai fait 4 US (Create, Read, Update, Delete).
J'ai vendu cette façon de faire aux autres apprenants, mais j'ai vu, que moi-même, je n'arrivai pas à la respecter.
Avec ce retour d'expérience, je rédigerai plutôt une US, pour une petite fonctionnalité :
L'ajout d'un film dans la base par exemple qui contiendrait le bouton dans l'IHM, la logique, et les méthodes associées dans l'API). (O)
